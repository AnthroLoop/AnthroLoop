# 意思決定フレームワーク

## 権限レベル

### Level 1: Auto-Approve（自律実行）
AI役職が承認なしで即座に実行できる。

- コードスタイル修正、リンター設定
- 日常KPIレポート生成
- 既存テンプレートに沿ったドキュメント更新
- バグ修正（既存機能の範囲内）
- テスト追加・修正

### Level 2: Propose & Execute（提案後実行）
担当役職が提案し、関連役職の確認後に実行。

- 軽微な機能改善（UI調整、コピー変更）
- 承認済み戦略内のコンテンツ作成
- 既存アーキテクチャ内のリファクタリング
- マーケティングコンテンツ（承認済みブランドガイドライン内）
- 月額1万円以下の支出

### Level 3: HILP Required（人間承認必須）
CPO(人間)の明示的承認が必要。

- 新機能の企画・開発着手
- 価格設定・変更
- 月額1万円超の予算コミット
- 外部サービス契約・解約
- ドメイン購入、インフラ選定
- ユーザー向け重要告知
- データスキーマの破壊的変更

### Level 4: CPO Only（人間のみ）
CPOが直接判断・実行する。

- 会社戦略の根本変更
- プロダクトビジョン策定
- 法務判断（利用規約、プライバシーポリシー）
- 資金調達・パートナーシップ
- 組織構造の変更

---

## RACI 運用ルール

1. **R(Responsible)**: タスクを実行する役職。出力物の品質に責任を持つ。
2. **A(Accountable)**: 最終承認者。Rの出力を承認/却下する権限。**1つの意思決定に対しAは必ず1名。**
3. **C(Consulted)**: 事前に意見を求める役職。Rは実行前にCの意見を確認する。
4. **I(Informed)**: 結果を事後報告する役職。

### 対立解消プロトコル
1. 関連役職間で議論（各自の根拠を数値で提示）
2. CEO が調整を試みる
3. 解決しない場合 → `/board` で全役職議論
4. それでも解決しない場合 → HILP エスカレーション（CPO判断）

## 意思決定記録

全てのLevel 3以上の意思決定は以下の形式で記録する:

```markdown
## Decision: [タイトル]
- **日付**: YYYY-MM-DD
- **レベル**: Level 3 / Level 4
- **提案者**: [役職]
- **承認者**: CPO
- **決定内容**: [概要]
- **根拠**: [データ・分析]
- **却下した代替案**: [WHY NOT]
- **影響範囲**: [部門・KPI]
```

記録場所: 各管轄の `docs/` 配下、またはADRとして `docs/architecture/adr/`

# ポートフォリオ戦略

> SSOT: コングロマリット戦略・ポートフォリオ方針はこのファイルが唯一の情報源。

---

## 目的

AnthroLoopはコングロマリット戦略を採用し、複数の事業を運営する。

### 戦略的根拠

1. **リスク分散**: 単一事業への依存を避け、異なる市場サイクル・顧客セグメントに分散投資する
2. **シナジー効果**: 事業間で技術・顧客基盤・オペレーションを共有し、個別運営より高い資本効率を実現する

---

## PMFゲート（厳格）

**1つ目の事業がPMFを達成するまで、2つ目の事業立ち上げは禁止。**

### PMF達成条件（全て満たすこと）

| 指標 | 閾値 | 根拠 |
|------|------|------|
| MRR | ≥ ¥100,000 | `.claude/rules/revenue-targets.md` Phase 1目標 |
| NRR | > 100% | 既存顧客からの収益が維持・拡大している |
| Churn Rate | < 5%/月 | 顧客が定着している |

### ゲート通過プロセス

1. CFOが上記3指標を3ヶ月連続で達成していることを確認
2. CEOがPMF達成レポートを作成し `/board` で共有
3. CPOがPMFゲート通過を承認（HILP Level 3）
4. 承認後、新規事業パイプラインの Validation 段階への進行が解禁

---

## 事業間シナジーの定義

新規事業提案時、以下6軸でシナジースコアを算出する。各軸1-5点、最低12/30で承認可能。

| 軸 | 定義 | 評価基準 |
|----|------|---------|
| **技術共有** | 既存の技術基盤・インフラを再利用できるか | 5: 80%以上再利用 / 1: 全く新規 |
| **顧客基盤共有** | 既存顧客にクロスセルできるか | 5: 同一ペルソナ / 1: 完全に別セグメント |
| **ブランド拡張** | AnthroLoopブランドが新事業に正の影響を与えるか | 5: 強いブランド移転 / 1: 無関係 |
| **データ活用** | 既存事業のデータが新事業の競争優位になるか | 5: データがコア資産 / 1: データ不要 |
| **オペレーション共有** | 既存の運用プロセス・ツールを再利用できるか | 5: 80%以上共有 / 1: 全く新規 |
| **クロスセル** | 事業間で相互送客できるか | 5: 自然な導線 / 1: 導線なし |

## 新ロール設計概要

### Auditor（外部顧問/監査）— Phase 1+

- **WHY**: 同じAIが意思決定と評価をする構造的盲点を補う。CPOの監視負荷を軽減
- **WHAT**: 四半期ガバナンス監査、財務サニティチェック、リスクレジスタ管理、`/board`でのDevil's Advocate
- **Phase 1（軽量）**: ガバナンス監査、リスク検出、バイアスチェック
- **Phase 2（フル）**: リスクレジスタ、年次監査

### CHRO — Phase 2+

- **WHY**: ロール数が8+になると、一貫性・重複・ギャップの管理が複雑化する
- **WHAT**: AIロールレジストリ管理、プロンプト品質レビュー、オンボーディング手順、組織健全性チェック

### BU Lead（事業責任者）— Phase 2+

- **WHY**: CEOが複数事業のボトルネックになるのを防ぐ。事業P&Lの明確な責任者
- **WHAT**: 担当事業のフルライフサイクル管理、共有サービスとの連携、事業レベルOKR管理
- **形式**: パラメータ化スキル（事業名を引数に取る）

### 不採用ロール

- **CSO（戦略参謀）**: WHY NOT — CEOの責任範囲と重複（SRP違反）。事業数が少ない段階では不要
- **CDO（データ責任者）**: Phase 4でプロダクトがデータ集約型の場合のみ評価

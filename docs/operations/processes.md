## インシデント対応

### 定義
- **P0**: サービス全停止、データ漏洩 → 即時対応
- **P1**: 主要機能停止 → 4時間以内に対応
- **P2**: 一部機能影響 → 24時間以内に対応
- **P3**: 軽微な不具合 → 次スプリントで対応

### フロー
1. 検知 → CTO が影響範囲を評価
2. P0/P1 → CPO に即時 HILP エスカレーション
3. 対応 → CTO が修正、COO が影響報告
4. 振り返り → 原因分析、再発防止策をSOP追加

## HILP外部通知プロセス

### フロー
1. AIがLevel 3以上のHILPを検出
2. `scripts/hilp-escalate.sh` が Slack + LINE + macOS に通知
3. CPOがSlackスレッドで応答
4. スクリプトが応答を取得 → Claude Code に返却
5. AIが応答に基づき業務継続

### コンポーネント
| コンポーネント | 用途 | 設定場所 |
|--------------|------|---------|
| `scripts/hilp-escalate.sh` | 通知送信・応答ポーリング | プロジェクトルート |
| Slack Bot | 送受信チャネル | `.env` (SLACK_*) |
| LINE Bot | モバイル通知 | `.env` (LINE_*) |
| `.claude/settings.local.json` | 権限・フック設定 | プロジェクト設定 |

### SLA
- 通知送信: HILP検出から30秒以内
- ポーリング間隔: 60秒
- 最大待機: 20分
- タイムアウト後: 会話内フォールバック

## インシデント対応

### 定義
- **P0**: サービス全停止、データ漏洩 → 即時対応
- **P1**: 主要機能停止 → 4時間以内に対応
- **P2**: 一部機能影響 → 24時間以内に対応
- **P3**: 軽微な不具合 → 次スプリントで対応

### フロー
1. 検知 → CTO が影響範囲を評価
2. P0/P1 → CPO に即時 HILP エスカレーション
3. 対応 → CTO が修正、COO が影響報告
4. 振り返り → 原因分析、再発防止策をSOP追加

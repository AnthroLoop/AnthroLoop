# 情報分類ポリシー

> **SSOT**: 情報分類に関する唯一の情報源。管轄: Auditor。

---

## 分類定義

| レベル | 定義 | アクセス範囲 | デフォルト |
|--------|------|-------------|-----------|
| **Public** | 外部公開可能な情報 | 制限なし | **デフォルト** |
| **Internal** | 組織内部でのみ共有する情報 | AI役職 + CPO | — |
| **Confidential** | 厳格なアクセス制限が必要な情報 | 管轄役職 + CPO | — |

> **原則**: デフォルトはPublic。OSSファースト・透明性重視の方針と整合させる。

---

## 分類別の具体例

### Public

| カテゴリ | 例 |
|---------|-----|
| ソースコード | `src/` 配下のOSSコード |
| ブランド | ブランドガイドライン、ロゴ使用規約 |
| コンテンツ | 技術ブログ記事、チュートリアル |
| 制度概要 | ファントムストック制度の概要・仕組み説明 |
| ドキュメント | 公開API仕様、コントリビューションガイド |

### Internal

| カテゴリ | 例 |
|---------|-----|
| 戦略 | OKR、戦略ドキュメント（`docs/strategy/`） |
| ガバナンス | RACI/HILPルール、SKILL.md |
| 会議体 | Board議事録、スプリント記録 |
| オペレーション | 内部プロセス定義、自動化スクリプト設計 |
| マーケティング | 未公開キャンペーン計画、競合分析 |

### Confidential

| カテゴリ | 例 |
|---------|-----|
| 財務 | 財務実績データ、P&L詳細、バリュエーション |
| セキュリティ | APIキー、認証情報、シークレット |
| 個人情報 | ユーザー個人情報、貢献者の個人連絡先 |
| エクイティ | ファントムストック個別契約、付与台帳 |
| 事業計画 | 未発表事業計画、M&A検討資料 |

---

## 分類原則

1. **デフォルトPublic**: 明示的に分類されない限りPublicとして扱う
2. **最小権限**: 必要最小限のアクセス範囲を適用する
3. **迷ったら格上げ**: 分類に迷った場合は上位レベルに分類する
4. **コードは原則Public**: `src/` 配下のコードはPublicを基本とする（シークレットは環境変数で管理）

## 分類変更ワークフロー

### 格上げ（Public → Internal）

| 項目 | 内容 |
|------|------|
| 申請者 | 管轄役職 |
| 承認者 | Auditor |
| 意思決定レベル | Level 2（提案後実行） |
| 記録 | 変更ログに記載 |

### 格上げ（Internal → Confidential）

| 項目 | 内容 |
|------|------|
| 申請者 | 管轄役職 |
| 承認者 | CPO |
| 意思決定レベル | Level 3（HILP必須） |
| 記録 | HILP Resolution + 変更ログ |

### 格下げ（全パターン）

| 項目 | 内容 |
|------|------|
| 申請者 | 管轄役職 |
| 承認者 | CPO |
| 意思決定レベル | Level 3（HILP必須） |
| 理由 | 格下げは不可逆（一度公開した情報は回収不能）のため |
| 記録 | HILP Resolution + 変更ログ |

---

## Auditorの情報分類統括責任

### Level 1（通常業務・自律実行）
- 新規ドキュメントへの分類ラベル付与
- 分類ラベルの整合性チェック
- 分類違反の検出・報告

### Level 2（提案後実行）
- 格上げ承認（Public → Internal）
- 分類ガイドラインの軽微な更新
- 四半期棚卸しの実施・レポート

### Level 3（HILP必須）
- 格上げ承認（Internal → Confidential）→ CPOへエスカレーション
- 格下げ承認（全パターン）→ CPOへエスカレーション
- ポリシーの重大改訂 → CPOへエスカレーション
- 新分類レベルの追加 → CPOへエスカレーション

---

## 定期レビュー

| レビュー | 頻度 | 担当 | 内容 |
|---------|------|------|------|
| 棚卸し | 四半期 | Auditor | 全ドキュメントの分類確認、不整合の修正 |
| ポリシーレビュー | 半期 | Auditor + CPO | ポリシー自体の有効性検証、必要に応じ改訂 |
